## NAT（Network Address Translation）

### なぜプライベートIPでインターネットに繋がれるのか

ルーターが「プライベートIP ↔ グローバルIP」を変換する翻訳機の役割を担う。

```
【PC】    192.168.1.2 ─┐
【スマホ】192.168.1.3 ─┤→ 【ルーター】→ インターネット
【TV】    192.168.1.4 ─┘   グローバルIP: 203.0.113.5
```

### NATの動き

```
① PC（192.168.1.2）が「Googleに繋いで」とルーターに送る
② ルーターが送信元をグローバルIP（203.0.113.5）に書き換えて送信
③ Googleはルーターのグローバルに返信
④ ルーターが「これはPCへの返事」と判断してPCに転送
```

## ポート番号

### IPアドレスだけでは足りない理由

IPアドレスが「建物の住所」なら、ポート番号は「部屋番号」。

ルーターは通信ごとにポート番号を割り当て、返信がどの機器宛かを判断する。

```
203.0.113.5:50001 ← PCのGoogleアクセス
203.0.113.5:50002 ← スマホのYouTubeアクセス
```

### よく使うポート番号（ウェルノウンポート）

| ポート | プロトコル | 用途 |
|---|---|---|
| **80** | HTTP | 通常のWeb通信 |
| **443** | HTTPS | 暗号化されたWeb通信 |
| **22** | SSH | サーバーへのリモート接続 |
| **3306** | MySQL | データベース接続 |
| **5432** | PostgreSQL | データベース接続 |

## TCP と UDP

### TCP — 確実に届ける方式

```
送信側          受信側
  │─「送っていいですか？」→│
  │←「どうぞ」───────────│  ← 3ウェイハンドシェイク
  │─「了解」─────────────→│
  │─ データ ──────────────→│
  │←「届きました」─────────│
```

- 届いたか確認しながら送る・届かなければ再送
- **遅いが確実**
- 用途：Web・メール・ファイル転送

### UDP — 速度優先の方式

```
送信側          受信側
  │─ データ ──────────────→│
  │─ データ ──────────────→│ ← 確認なし・再送なし
```

- **速いが不確実**
- 用途：動画・オンラインゲーム・ビデオ通話

### 比較表

| | TCP | UDP |
|---|---|---|
| 信頼性 | 高い（確認あり） | 低い（確認なし） |
| 速度 | 遅め | 速い |
| 用途 | Web・メール・ファイル転送 | 動画・ゲーム・通話 |
